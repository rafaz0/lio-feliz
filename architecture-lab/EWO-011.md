## Objetivo

Materializar o Bloco A da PI-011 (Platform Consolidation & Product Readiness) através da implementação incremental dos 2 módulos de Comercialização:

1. **Assinaturas (19)** — Novo módulo: planos, assinaturas, autorização por plano e billing simulado
2. **Perfil do Investidor (20)** — Novo módulo: questionário de risco, classificação, integração com carteira

Esta Engineering Work Order transforma a especificação arquitetural aprovada em uma sequência rastreável de Slices de implementação, preservando integralmente as Frozen Baselines.

## ADRs Registradas (REC-011-001)

### ADR-011-001: Extensão do ISubscriptionRepository (Decisão O1)

| Campo         | Valor                                                                                                                                            |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| Contexto      | O módulo 19 requer CRUD de planos e assinaturas. Já existe `ISubscriptionRepository` com `ObterPlanoAtivo`, `Salvar`, `ListarPlanosDisponiveis`. |
| Decisão       | Estender o port existente com 5 novos métodos em vez de criar um novo port.                                                                      |
| Consequências | Métodos existentes preservados. Implementações existentes (Fake + Supabase) precisam ser estendidas. Nenhum código cliente precisa ser alterado. |

### ADR-011-002: Extensão do IConfigurationRepository (Decisão O2)

| Campo         | Valor                                                                                                                                                        |
| ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Contexto      | O módulo 22 requer persistência de UserPreferences, Theme e DashboardLayout. Já existe `IConfigurationRepository` com `ObterEstrategia`, `SalvarEstrategia`. |
| Decisão       | Estender o port existente para cobrir UserPreferences. Postergado para EWO-012 (Bloco B).                                                                    |
| Consequências | Nenhuma ação imediata. Registrado para rastreabilidade futura.                                                                                               |

### ADR-011-003: Onboarding sem Infraestrutura Própria (Decisão O3)

| Campo         | Valor                                                                                                          |
| ------------- | -------------------------------------------------------------------------------------------------------------- |
| Contexto      | O módulo 21 (Onboarding) é puramente orquestração — consome conteúdo do módulo 17 e armazena estado.           |
| Decisão       | Não criar port de infraestrutura para Onboarding. Consumir `IGlossaryRepository` e `IConfigurationRepository`. |
| Consequências | Zero novos ports. Onboarding puramente Application + Presentation. Nenhuma ação nesta EWO (Bloco B).           |

## Estratégia Geral

### Ordem de Dependências

```
BR + Core Domain (documentação + entidades)
        ↓
Application Contracts (Commands, Queries, DTOs, Ports)
        ↓
Infrastructure Adapters (Supabase + InMemory)
        ↓
Presentation Feature (components, hooks, viewmodels, tests)
```

Esta ordem se repete para cada módulo, respeitando a sequência: **Assinaturas (19) → Perfil do Investidor (20)**.

### Dependências entre Módulos

```
19 — Assinaturas (independente — não depende de Perfil)
        ↓
20 — Perfil do Investidor (independente — não depende de Assinaturas)
```

### Estratégia de Testes

- **Testes unitários (Vitest):** obrigatórios, cobertura >90% do código novo
- **Testes de integração:** quando a Slice toca fronteiras (Ports, Dispatcher)
- **Testes de arquitetura:** verificar Dependency Rule e Architecture Guard para as novas features
- **Fakes:** implementações InMemory utilizadas desde a Slice de Infra
