## 1. Objetivo

Esta Engineering Review analisa a PI-004 (Modelo Patrimonial do Lio Feliz) para:

- Interpretar corretamente as decisões arquiteturais aprovadas

## 4. Avaliação de Regressão

### 4.1 Compatibilidade com Core Foundation (GOV-006)

- ✅ **AggregateRoot pattern** já estabelecido — Portfolio deve estender `AggregateRoot<PortfolioId>`
- ✅ **DomainEvent pattern** já estabelecido — FinancialEvent deve estender `DomainEvent`
- ✅ **ValueObjects** (Ticker, Quantity, Money, PortfolioId) prontos para uso
- ✅ **EntityId** base para identificadores
- ✅ **Result Pattern** para operações que podem falhar

### 4.2 Impacto no Código Existente (`src/lib/portfolio/`)

| Arquivo | Impacto | Estratégia |
|---------|---------|------------|
| `models.ts` | Interfaces TS → Value Objects DDD | Migração gradual; manter compatibilidade |
| `consolidator.ts` | Lógica funcional → Portfolio.consolidate() | Substituir por método do Aggregate Root |
| `history.ts` | Função pura → Projection.rebuild() | Migrar para Projection layer |
| `asset-types.ts` | Utilitário compartilhado | Manter como shared kernel |

**Risco de regressão:** Baixo. O código legado em `lib/portfolio` é usado pelas rotas atuais. A migração deve ser feita mantendo a API pública (server functions) estável enquanto a implementação interna migra para o novo domínio.

## 6. Observações e Melhorias

### 6.1 Decisões de Implementação (delegadas à EWO-002)

1. **Persistência:** PI-004 não define (DA-010). EWO-002 deve decidir: Event Store? Relacional? Híbrido?
2. **FinancialEvent.correlationId:** Usar para rastrear operações relacionadas (ex: buy+sell same day)
3. **Portfolio.multiplePortfolios:** Arquitetura suporta (DA-003, I-004), mas UI inicial pode expor apenas 1
4. **Rebuild strategy:** Lazy (on-demand) vs Eager (materialized views) — definir na EWO
5. **Snapshot/Checkpoint:** Para performance de rebuild de histórico longo

### 6.2 Melhorias Identificadas

- **DA-012 (Linguagem Ubíqua):** Renomear `Operation` → `FinancialEvent` no domínio; manter `Operation` apenas na camada de aplicação/DTO
- **Position como Value Object:** Imutável, comparável estruturalmente, rebuildable
- **Projection Interface:** `Projection.rebuild(events: FinancialEvent[]): Projection` — contract único
- **Event Versioning:** FinancialEvent deve ter `version` para evolução futura
