## Objetivo

Materializar o Bloco A da PI-012 — Completude da Presentation Layer — implementando a camada de apresentação dos 4 módulos pendentes:

1. **Backtests (14)** — BacktestsPage, StrategyForm, BacktestRunner, gráficos, hooks, Composition Root
2. **Alertas (15)** — AlertsPage, AlertCard, AlertRuleForm, hooks, Composition Root
3. **Educação (17)** — EducationPage, GlossaryList, TooltipProvider, hooks, Composition Root
4. **Exportação Avançada (18)** — AdvancedExportPage, ExportTemplateCard, ExportJobList, hooks, Composition Root

Nenhum Core Domain, Application ou Infrastructure é alterado. Apenas Presentation pura (R-018).

---

## Escopo

### Inclui:

- 4 features Presentation em `features/backtests/`, `features/alerts/`, `features/education/`, `features/advanced-export/`
- Componentes, hooks (TanStack Query + Dispatcher), viewmodels (mappers puros)
- Estados: loading, empty, error
- Testes de componente + architecture test R-10
- Extensão do Composition Root por adição
- Atualização documental

### Não contempla:

- Modificação de Core Domain (entidades, VOs, domain services, errors)
- Modificação de Application Layer (commands, queries, services, ports, DTOs)
- Modificação de Infrastructure Layer (repositories, gateways)
- Criação de novos ports
- Criação de novos commands ou queries

---

## Princípios Obrigatórios

1. **Presentation Pura (R-018)** — Nenhum arquivo de Core, Application ou Infrastructure é criado ou modificado.
2. **Feature-First (PA-008-004 / R-006)** — Cada módulo em `features/<feature>/`.
3. **Dispatcher Only** — Hooks usam `useDispatcher()` → `IDispatcher`.
4. **Reuso máximo** — Commands, Queries, DTOs, Ports existentes. Nada novo nessas camadas.
5. **Frozen Baselines** preservadas.
6. **Testes dentro da Slice** — Nenhuma Slice é concluída sem seus testes.

---

## Estratégia Geral

### Ordem de Execução

```
Backtests 14 → Alertas 15 → Educação 17 → Exportação 18
```

A ordem segue a complexidade crescente das features e independência entre módulos.

### Estrutura por Módulo

Cada módulo segue 3 Slices:

1. **Slice A — Components:** Páginas, cards, formulários, estados (loading/empty/error)
2. **Slice B — Hooks + ViewModels:** TanStack Query hooks + mappers puros
3. **Slice C — Composition Root + Tests:** Registro no dispatcher + testes de componente + R-10
