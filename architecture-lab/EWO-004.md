## Objetivo

Materializar a arquitetura definida pela PI-006 através da implementação incremental da Infrastructure Layer do Lio Feliz.

Esta Engineering Work Order transforma a especificação arquitetural aprovada em uma sequência rastreável de Slices de implementação, preservando integralmente os contratos definidos pela arquitetura.

A implementação deverá produzir adaptadores concretos de infraestrutura para os 10 Ports da Application Layer, utilizando Supabase/PostgreSQL para persistência, gateways para dados externos, e fakes in-memory para testes.

---

## Slices de Implementação

### Slice 1 — Fakes + Estrutura de Diretórios

**Objetivo:** Criar o diretório `src/infrastructure/` com suas subpastas e implementar todas as 10 classes Fake (in-memory) para substituir os mocks inline nos testes existentes.

**Arquivos a criar:**

| Arquivo | Descrição |
|---------|-----------|
| `src/infrastructure/index.ts` | Barrel |
| `src/infrastructure/fakes/fake-portfolio-repository.ts` | Fake de IPortfolioRepository |
| `src/infrastructure/fakes/fake-asset-repository.ts` | Fake de IAssetRepository |
| `src/infrastructure/fakes/fake-configuration-repository.ts` | Fake de IConfigurationRepository |
| `src/infrastructure/fakes/fake-subscription-repository.ts` | Fake de ISubscriptionRepository |
| `src/infrastructure/fakes/fake-projection-repository.ts` | Fake de IProjectionRepository |
| `src/infrastructure/fakes/fake-data-gateway.ts` | Fake de IDataGateway |
| `src/infrastructure/fakes/fake-unit-of-work.ts` | Fake de IUnitOfWork |
| `src/infrastructure/fakes/fake-domain-event-publisher.ts` | Fake de IDomainEventPublisher |
| `src/infrastructure/fakes/fake-notification-port.ts` | Fake de INotificationPort |
| `src/infrastructure/fakes/fake-import-interpreter.ts` | Fake de IImportInterpreterPort |

**Cada Fake deve:**
- Implementar a interface do Port completamente
- Usar estruturas in-memory (Map, Array) para armazenamento
- Incluir método `reset()` para limpar estado entre testes
- Ser exportado via barrel `src/infrastructure/fakes/index.ts`

**Testes:** Nenhum específico para fakes (são utilizados pelos testes existentes da Application Layer).

**Critério de Aceitação:**
- `src/infrastructure/` criado com todas as subpastas
- 10 classes Fake implementadas e exportadas
- `src/infrastructure/fakes/index.ts` barrel funcional

## Dependências entre Slices

```
S1 (Fakes) → S2 (Repositories CRUD)
                         ↓
              S3 (Subscription + Projection)
                         ↓
              S4 (UoW + Events) ──────→ S5 (Gateways)
                         ↓
              S6 (Notifications + Interpreter)
                         ↓
              S7 (Migration + Integration)
```

S1 não depende de nenhum slice anterior e pode ser executado primeiro.
S7 depende de todos os slices anteriores.
