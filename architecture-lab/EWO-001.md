# EWO-001 — Implementação do Núcleo Arquitetural (Core Foundation) ✅ CONCLUÍDO

**Base de Engenharia:** PI-001 v1.0, PI-002 v1.0, PI-003 v1.0

**Objetivo:** Materializar a Core Foundation (C-001) e o modelo de domínio inicial (C-002) no repositório oficial, conforme lições aprendidas na AIR-001, GIT-FORENSICS-001 e ER-C002-001.

**Escopo:** Implementação exclusiva do núcleo DDD tático e das primeiras entidades canônicas. Não inclui Interpretation Layer (PI-001), fluxo de eventos (PI-003) ou persistência.

**Restrições:**
- Nenhum `throw` no domínio — usar `Result`
- `validate()` pós-construtor (lição ER-C002-001)
- Nenhuma dependência de infraestrutura,框架 ou banco
- Nenhum `any` no código de domínio
- Toda slice deve ser concluída com commit e push independentes

## Slice 03 — Entity + EntityId

| Campo | Valor |
|---|---|
| Objetivo | `Entity<TId>` com identidade por `EntityId` abstrato |
| PIs base | —
| Arquivos | `src/core/domain/entity.ts`, `src/core/domain/entity-id.ts` |

**EntityId:** A estratégia de geração de identificadores permanece desacoplada da implementação concreta. `EntityId` deve permitir substituição futura do mecanismo de geração (UUIDv7 é a implementação concreta inicial, não um vínculo arquitetural).
