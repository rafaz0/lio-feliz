# Objetivo

Este documento registra ideias, conceitos, intenções e possibilidades para o futuro do Lio Feliz.

Diferentemente da documentação oficial, este arquivo não define regras obrigatórias.

Seu objetivo é preservar o raciocínio desenvolvido ao longo da evolução do projeto para que nenhuma ideia importante seja perdida.

As ideias registradas aqui poderão, futuramente, tornar-se funcionalidades oficiais, documentos técnicos ou decisões arquiteturais.

---

# Filosofia Central

O Lio Feliz não é apenas um sistema para acompanhar investimentos.

Seu objetivo é organizar completamente a vida financeira do investidor de longo prazo.

Sempre que uma nova funcionalidade for proposta, a pergunta deverá ser:

> Esta funcionalidade ajuda o usuário a organizar seu patrimônio de forma mais simples, segura e eficiente?

Se a resposta for negativa, sua inclusão deve ser cuidadosamente avaliada.

---

# O Problema que o Lio Feliz Resolve

Hoje o investidor costuma utilizar várias plataformas ao mesmo tempo.

Exemplo:

- Corretora;
- B3;
- Investidor10;
- Planilhas;
- Aplicativos de controle;
- Calculadoras para Imposto de Renda.

Isso gera retrabalho, inconsistências e perda de tempo.

O Lio Feliz pretende concentrar toda essa gestão em um único ambiente.

# Princípios Fundamentais

## O usuário é dono dos próprios dados.

Nenhuma integração externa será obrigatória.

O sistema deverá funcionar de forma totalmente independente.

---

## Integrações apenas facilitam o trabalho.

As integrações não serão a base do sistema.

Elas apenas automatizam processos.

---

## O sistema deve reduzir trabalho manual.

Sempre que for possível automatizar um processo com segurança, essa será a solução preferencial.

---

## Transparência

O usuário deverá compreender como cada cálculo foi realizado.

O sistema deverá explicar seus resultados sempre que possível.

# Ideias por Tema

## Dashboard

Objetivo:

Ao abrir o sistema, o usuário deve conseguir responder rapidamente:

- Quanto possuo hoje?
- Quanto recebi este mês?
- Minha carteira está balanceada?
- Preciso realizar algum aporte?
- Quanto falta para atingir minha meta?
- Existem eventos corporativos pendentes?
- Existe alguma pendência fiscal?

---

## Rebalanceamento

O rebalanceamento será um dos principais diferenciais do projeto.

O sistema deverá indicar:

- percentual atual;
- percentual alvo;
- diferença;
- quanto investir em cada ativo;
- impacto do aporte sugerido;
- carteira após o aporte.

As recomendações deverão respeitar integralmente a estratégia definida pelo usuário.

O sistema não recomendará ativos com base em opiniões ou análises de mercado.

---

## Integração com a B3

Objetivo ideal:

Após uma compra realizada na corretora, o ativo deverá aparecer automaticamente na carteira do usuário.

Caso a sincronização não seja possível, o sistema deverá oferecer alternativas como importação por CSV.

A ausência da integração nunca poderá impedir o funcionamento do sistema.

---

## Portabilidade do Investidor10

O Investidor10 será utilizado apenas para facilitar a migração do usuário para o Lio Feliz.

Após a importação, o sistema utilizará exclusivamente seus próprios dados.

O funcionamento do projeto nunca dependerá do Investidor10.

---

## Gestão Fiscal

A organização fiscal deverá ocorrer continuamente durante todo o ano.

O objetivo é que, ao chegar o período da declaração do Imposto de Renda, praticamente todas as informações já estejam prontas.

Sempre que possível, relatórios deverão ser gerados automaticamente.

---

## Eventos Corporativos

Sempre que possível, os seguintes eventos deverão ser processados automaticamente:

- Dividendos;
- Juros sobre Capital Próprio (JCP);
- Rendimentos de FIIs;
- Bonificações;
- Desdobramentos;
- Grupamentos;
- Subscrições;
- Amortizações;
- Incorporações;
- Fusões;
- Cisões;
- Conversões de ativos.

---

## Ativos Internacionais

O projeto deverá tratar ativos internacionais como cidadãos de primeira classe.

Não serão adaptações feitas posteriormente.

Todo o sistema deverá ser preparado para lidar com:

- múltiplas moedas;
- câmbio;
- ações estrangeiras;
- ETFs internacionais;
- REITs;
- tributação específica;
- diferentes calendários de mercado.

---

## Objetivos Financeiros

O usuário poderá definir objetivos como:

- patrimônio total;
- renda passiva mensal;
- aposentadoria;
- metas por classe de ativos.

O sistema deverá acompanhar automaticamente a evolução dessas metas.

---

## Filosofia de Crescimento

Sempre que novas funcionalidades forem implementadas, deverá ser avaliado:

- se realmente resolvem um problema;
- se aumentam a complexidade;
- se podem ser incorporadas a funcionalidades existentes;
- se respeitam a simplicidade do sistema.
